ИНСТРУКЦИЯ ЗАПУСКА ПРОЕКТА НА ДРУГОМ ПК

Проект: МебельОрг
GitHub: https://github.com/v45279729/1


1. ЧТО НУЖНО УСТАНОВИТЬ

На другом ПК должны быть установлены:

- Python 3.11 или 3.12
- MySQL Server 8.0
- MySQL Workbench, желательно
- Git, если проект будут скачивать с GitHub


2. СКАЧАТЬ ПРОЕКТ С GITHUB

Открыть PowerShell и выполнить:

git clone https://github.com/v45279729/1.git
cd 1


3. СОЗДАТЬ ВИРТУАЛЬНОЕ ОКРУЖЕНИЕ И УСТАНОВИТЬ БИБЛИОТЕКИ

В папке проекта выполнить:

python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt


4. НАСТРОИТЬ ПОДКЛЮЧЕНИЕ К MYSQL

Создать файл .env на основе примера:

copy .env.example .env
notepad .env

В файле .env указать данные MySQL на этом ПК:

MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=пароль_от_mysql
MYSQL_DATABASE=mebelorg

Важно:
Файл .env не загружается в GitHub специально, потому что в нем хранится пароль.
На каждом ПК его нужно создавать вручную.


5. СОЗДАТЬ БАЗУ ДАННЫХ И ИМПОРТИРОВАТЬ ДАННЫЕ

MySQL Server должен быть запущен.

Выполнить:

python scripts\import_data.py

Скрипт создаст базу mebelorg, таблицы и загрузит исходные данные проекта.


6. ЗАПУСТИТЬ ПРИЛОЖЕНИЕ

Выполнить:

python -m app.main

Или после настройки можно запускать двойным кликом:

import_data.bat
run_app.bat

8. ЕСЛИ ПРОЕКТ НЕ ЗАПУСКАЕТСЯ

Проверить, что активировано виртуальное окружение:

.venv\Scripts\activate

Проверить, что установлены библиотеки:

pip install -r requirements.txt

Проверить, что MySQL Server запущен.

Проверить пароль в файле .env.

Повторить импорт:

python scripts\import_data.py

После этого снова запустить:

python -m app.main
